## Protocol Design Principles

- Protocols are living documents.
- Test before standardizing.
- Every protocol must have at least one documented execution example.
- Improve protocols through retrospectives.
- Capture decisions, not conversations.

## Decision-Making Principles

- Build credibility through evidence, not claims.
- Build the smallest implementation that preserves the long-term architecture.
- Reduce scope before reducing quality.
- Invest after validation.
