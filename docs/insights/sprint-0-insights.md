# Sprint 0 Insights

## Insight 001 — Learn the engineering workflow before automating it

### Observation

Building an AI-assisted engineering workflow starts with learning the engineering workflow itself.

During Sprint 0, I intentionally practiced the complete Git workflow instead of delegating everything to AI.

### Why it matters

AI can accelerate software engineering, but it cannot replace understanding of the engineering process.

Understanding branching, pull requests, merge conflicts, reviews, and deployment makes AI collaboration significantly more effective.

### Evidence

Sprint 0 included the complete Git workflow:

- Repository creation
- Clone
- Feature branch
- Commit
- Push
- Pull Request
- Merge
- Merge conflict resolution
- Branch cleanup

### Reusable takeaway

Before optimizing a workflow with AI, understand the workflow itself.
