# ADR-NNN — <short decision title>

**Status:** Proposed | Accepted | Superseded by ADR-XXX
**Date:** YYYY-MM-DD

## Context

The forces at play: the situation, the relevant constraints, and why a decision is needed now. Cite sources (`PROJECT.md` §, other ADRs, discovery report). Keep to what's needed to understand the decision — not a history of the discussion.

## Decision

The single architectural decision, stated actively ("We will…"). Use numbered points only when they are different aspects of this one decision — if you're tempted to record a second, separate decision, write another ADR instead.

## Alternatives considered

The options not chosen, each with a one-line reason for rejection. This is often the most reusable part of an ADR — it records the roads not taken so a future reader doesn't reopen them blindly.

## Consequences

What becomes easier and what becomes harder as a result. No fixed categories — but always state the important trade-offs, so the record is never one-sided.

## Impacted Artifacts

Project artifacts that should reflect or respect this decision (e.g. `PROJECT.md`, `CLAUDE.md`, Architecture Design documents, other ADRs). Traceability only — not implementation guidance.
